## 详细修复记录

### P3-1: CraftingPlanScreen null 守卫缺失

**提交**: 9ac6738

**位置**: `CraftingPlanScreen.java:1221-1226`

**问题**: `preferenceKey` 返回值可能为 null，直接调用 `toString()` 会导致 NPE。

**修复**:
```java
ResourceLocation pk = CraftingResolver.preferenceKey(treeKey.stack(1));
if (pk != null) forced.remove(pk.toString());
```

---

### P3-2: 协议版本号未校验

**提交**: 9ac6738

**位置**: `PlanResponsePacket.java:363-368`

**问题**: 解码时未校验协议版本，可能导致不兼容客户端按错误布局解读。

**修复**:
```java
int version = buf.readVarInt();
if (version != CraftPlanGraph.CURRENT_VERSION) {
    throw new DecoderException("Unsupported graph protocol version: " + version
            + ", expected: " + CraftPlanGraph.CURRENT_VERSION);
}
```

---

### P3-3: 图标查找未缓存

**提交**: 9ac6738

**位置**: `RecipePreviewRenderer.java:286-293`

**问题**: 每帧渲染都查找 ForgeRegistries，属于渲染热路径。

**修复**:
```java
private net.minecraft.world.item.Item gemCuttingTableItem;

if (gemCuttingTableItem == null) {
    gemCuttingTableItem = ForgeRegistries.ITEMS
            .getValue(new ResourceLocation("apotheosis", "gem_cutting_table"));
}
```

### P3-4: RecipeGuiLayoutsMixin player null 检查缺失

**提交**: 3dedc9a

**位置**: `RecipeGuiLayoutsMixin.java:122-123`

**问题**: player 变量未做 null 检查，直接使用可能导致 NPE。

**修复**:

var player = Minecraft.getInstance().player;

### P3-6: 风格不一致的 cancel 调用

### P3-7: BackpackReflection 注解 modId 不一致

**提交**: 47186e9

**位置**: `BackpackReflection.java:7-8`

**问题**: 注解声明 modId 为 SOPHISTICATED_BACKPACKS，但实际处理两个 mod。

```java
@ModReflection(modId = ModIds.SOPHISTICATED_BACKPACKS + "+" + ModIds.BETTER_COMBAT,

```

**说明**: 仅元数据改进，无运行时影响。
