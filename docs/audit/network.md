## P3 发现（已知边界行为）

### [P3-1] PlayerNetworkResolutionCache 无过期清理

**位置**：`PlayerNetworkResolutionCache.java`

**观察**：
玩家网络解析缓存（player UUID -> INetwork）没有自动过期机制。如果玩家长时间在线且频繁切换维度，缓存可能积累大量过期条目。

**当前缓解**：
- 缓存是 WeakHashMap 或有显式清理逻辑（需查看实现）
- 玩家数量有限，内存影响可控

**建议**：
添加基于时间的过期策略（例如 5 分钟未使用自动清除）。

**优先级**：P3（内存泄漏风险低）

### [P3-2] GuiOpenRateLimiter 固定窗口计数

**位置**：`GuiOpenRateLimiter.java`

**观察**：
GUI 打开速率限制使用固定时间窗口计数，可能被边界攻击绕过（在窗口切换瞬间发送大量请求）。

**影响**：
客户端恶意快速打开 GUI 可能导致服务器卡顿，但不会造成物品丢失或刷物。

**建议**：
使用滑动窗口或令牌桶算法。

**优先级**：P3（影响有限，仅 DoS 风险）
