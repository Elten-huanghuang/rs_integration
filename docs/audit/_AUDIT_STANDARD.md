# RS-Integration 静态审计标准 (共享 brief)

本次为**纯静态审计**：只读代码、只记录问题，**绝不修改任何源文件**。

## 严重度分级
- **P0 阻断级**：物品复制/丢失、崩服/崩客、数据损坏、死锁、无限递归/栈溢出、SRG 重映射崩溃。
- **P1 高**：竞态条件、结果不正确、资源泄漏（区块 ticket / 未关闭句柄）、能力门控失效、异常被吞导致状态不一致。
- **P2 中**：健壮性缺陷（未守卫 cast、NPE 风险、边界处理缺失）、并发可见性问题、错误的比较器/排序。
- **P3 低**：死代码、重复逻辑、魔法数字、命名/注释、可维护性。

## 审计维度（逐文件对照）
1. **资源守恒**：抽取/退款/提交是否配对；副产物是否重复计入；abort/超时路径是否丢物或刷物。
2. **并发与线程安全**：共享可变状态是否有同步；ledger/账本操作原子性；跨线程访问 world/容器；async chain 生命周期。
3. **反射安全**：是否反射原版方法（SRG 会重映射导致运行时崩），是否有 fallback；`getMethod`/`getDeclaredMethod` 目标是否稳定。
4. **Mixin 正确性**：注入点、`cancellable`、`@WrapOperation`/`@Redirect` 副作用、是否可能改变原版语义、plugin 的 shouldApplyMixin 守卫。
5. **Null 与异常**：未守卫 cast、Optional 误用、`catch` 吞异常、错误恢复是否留下脏状态。
6. **递归/栈安全**：配方展开、树遍历、handler 回调是否有深度/环守卫。
7. **端隔离**：packet 处理是否在正确端、client-only 类是否泄漏到 server、`DistExecutor` 使用。
8. **算法正确性**：DNF 展开、预算门控、比较器、去重、批处理数量计算。
9. **代码质量**：死代码、重复、可维护性。

## 已知历史坑（重点复查是否复发/遗漏同类）
- 副产物字段扫描 + getOutputs() 含主产物 → 重复计入刷物。
- delegate 无过滤字段扫描取错产物；collectResult 单返回值吞多产物。
- 同名重载只有一个有递归守卫 → 栈溢出。
- 解析器 missing 侧超时/次数守卫挡在直接扣库存前 → 误报缺料。
- COMBINATION_OR 被 flatten 成 AND。
- 多步链前序产物在后续 abort 时被 clear → 丢物。
- 比较器排序标准用错。
- ForgeChunkManager 不 refcount，多 owner 共享 ticket。
- 能力门控只 warn 不拦截。

## 输出格式（每个 agent 写入自己的 docs/audit/<domain>.md）
每条发现：
```
### [P0|P1|P2|P3] 简短标题
- 文件: 相对路径:行号
- 维度: <维度名>
- 现象: 客观描述代码在做什么
- 风险: 会导致什么后果（给出触发场景）
- 证据: 关键代码片段（简短引用）
```
按严重度从高到低排列。文件顶部给出该域的文件清单与一句话总评。
只记录有**代码证据**的问题，不臆测；不确定的标注"待确认"。
